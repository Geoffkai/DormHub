# DormHub

DormHub is a Java desktop application for dormitory management. It manages residents, rooms, assignments, payments, and dorm passes in a Swing desktop UI.

The current repository contains the application source, its packaged runtime files, and an InstallForge-based Windows installer. The active app source lives under `Thumbler_sa_ref/DormHub/`.

## What DormHub Does

- Manage resident records
- Manage room information and availability
- Track room assignments and room occupancy
- Record payments and payment status
- Manage dorm pass requests
- Load database credentials from a local configuration file

## Requirements

To build or run the desktop app directly, you need:

- Windows 10 or later for the installer path
- A local MySQL Server instance
- The bundled libraries under `Thumbler_sa_ref/DormHub/libs/`
- A Java runtime if you are not using the installer

The installer already includes a bundled JDK/JRE folder, so end users do not need to install Java separately.

## Installer

The recommended way to run DormHub is to use the EXE installer:

- File: `Thumbler_sa_ref/DormHub/installers/DormHub-Setup.exe`
- Type: InstallForge Windows installer
- Behavior: installs the app, bundled runtime, libraries, and assets in one step
- Result: users can launch DormHub without manually setting up Java

The installer project file is stored in [DormHub-SetUp.ifp](DormHub-SetUp.ifp).

## Running From The Workspace

If you want to launch the app without the installer, start it from `Thumbler_sa_ref/DormHub/` so the app can find `app.env`, the runtime files, and the image assets.

```cmd
cd Thumbler_sa_ref\DormHub
java -cp "DormHub.jar;libs/*" com.dormhub.Main
```

## Database Setup

DormHub reads its database credentials from `Thumbler_sa_ref/DormHub/app.env` or `Thumbler_sa_ref/DormHub/resources/db.properties`, depending on how you launch it.

A minimal MySQL configuration looks like this:

```properties
db.url=jdbc:mysql://localhost:3306/dormhub?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
db.user=root
db.password=your_password

app.username=admin
app.password=admin123
```

If the database does not exist, the application now loads the SQL bootstrap script and creates the schema plus sample data automatically.

## Runtime Files

The app depends on these runtime files being present next to the executable or inside the packaged output:

- `DormHub.jar`
- `libs/`
- `resources/`
- `assets/img/` or `img/`
- `jdk-25.0.3+9-jre/` when using the installer bundle

The UI loads images from `/img/...`, and the runtime loader also falls back to `assets/img/...` when running directly from the workspace.

## Notes

- The application uses a layered architecture: View -> Controller -> Service -> DAO -> Database.
- Sample data and schema definitions live in [sql/dormhub.sql](sql/dormhub.sql).
- Keep database credentials and other sensitive values out of public repositories.
- The installer bundle is self-contained, so the EXE can be distributed without a separate Java install step.
